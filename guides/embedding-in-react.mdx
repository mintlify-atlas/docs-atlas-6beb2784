## Overview

Convex Panel provides a seamless way to embed a development panel directly into your React application. The panel gives you real-time access to logs, data inspection, function testing, and health monitoring.

## Installation

<Steps>
  <Step title="Install the package">
    Install Convex Panel as a development dependency:

    ```bash
    npm install convex-panel --save-dev
    ```
  </Step>

  <Step title="Run the setup command">
    From your project root (where your `convex/` folder lives), run:

    ```bash
    npx convex-panel setup
    ```

    This interactive command will:
    - Create or update `convex/http.ts` with OAuth endpoints
    - Prompt you for required environment variables
    - Configure your project for OAuth authentication
  </Step>

  <Step title="Configure environment variables">
    Add the following variables to your `.env` file:

    ```bash
    VITE_CONVEX_URL=https://your-deployment.convex.cloud
    VITE_OAUTH_CLIENT_ID=your-oauth-client-id
    VITE_CONVEX_TOKEN_EXCHANGE_URL=https://your-deployment.convex.site/oauth/exchange
    CONVEX_ACCESS_TOKEN=your-access-token
    ```
  </Step>

  <Step title="Set up OAuth in Convex Dashboard">
    1. Go to [Convex Dashboard](https://dashboard.convex.dev)
    2. Navigate to your team settings and create an OAuth application
    3. Copy the **Client ID** and **Client Secret**
    4. In your Convex deployment settings, add these environment variables:
       - `CONVEX_CLIENT_ID` - Your OAuth client ID
       - `CONVEX_CLIENT_SECRET` - Your OAuth client secret (never in frontend `.env`)
       - `OAUTH_REDIRECT_URI` - Your frontend origin (e.g., `http://localhost:5173`)
  </Step>
</Steps>

## Basic Usage

### Standard React + Vite

For React applications using Vite, use the `convex-panel/react` import:

```tsx src/App.tsx
import { ConvexReactClient, ConvexProvider } from "convex/react";
import ConvexPanel from "convex-panel/react";

const convex = new ConvexReactClient(import.meta.env.VITE_CONVEX_URL);

function App() {
  return (
    <ConvexProvider client={convex}>
      {/* Your app components */}
      <ConvexPanel />
    </ConvexProvider>
  );
}

export default App;
```

## Configuration Props

The `ConvexPanel` component accepts the following props:

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `convex` | `ConvexReactClient` | Auto-detected | Convex client instance (auto-detected from ConvexProvider) |
| `deployUrl` | `string` | Auto-detected | Your Convex deployment URL |
| `accessToken` | `string` | - | Manual access token (if not using OAuth) |
| `deployKey` | `string` | - | Deploy key for admin-level access |
| `defaultTheme` | `'dark' \| 'light'` | `'dark'` | Initial theme (persisted in localStorage) |
| `useMockData` | `boolean` | `false` | Use mock data for development/testing |
| `onError` | `(error: string) => void` | - | Error callback |

### Example with Custom Configuration

```tsx
<ConvexPanel
  defaultTheme="light"
  accessToken={process.env.CONVEX_ACCESS_TOKEN}
  onError={(error) => console.error("Panel error:", error)}
/>
```

## Advanced Configuration

### Custom OAuth Setup

For custom OAuth setups with your own backend endpoint:

```tsx
<ConvexPanel
  oauthConfig={{
    clientId: "your-client-id",
    redirectUri: window.location.origin,
    scope: "project", // or "team"
    tokenExchangeUrl: "https://your-deployment.convex.site/oauth/exchange",
  }}
/>
```

### Custom Authentication

Provide your own authentication implementation:

```tsx
import { useOAuth } from "convex-panel";

function App() {
  const customAuth = useOAuth(oauthConfig);
  
  return (
    <ConvexProvider client={convex}>
      <ConvexPanel auth={customAuth} />
    </ConvexProvider>
  );
}
```
