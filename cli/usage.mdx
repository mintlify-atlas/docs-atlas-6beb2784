## Command Overview

The Convex Panel CLI provides a single command for setting up your project:

```bash
convex-panel setup
```

This interactive command handles all configuration needed to get started with Convex Panel.

## CLI Entry Point

The CLI is implemented in `bin/convex-panel.js` and registered in package.json:174:

```json
"bin": {
  "convex-panel": "./bin/convex-panel.js"
}
```

## Command Structure

The CLI accepts a single command argument (bin/convex-panel.js:11):

```javascript
const command = process.argv[2];
```

### Valid Commands

- `setup` - Run the interactive setup process
- No argument - Display help message
- Any other argument - Display error and help message

## Setup Command

When you run `convex-panel setup`, it executes two scripts sequentially (bin/convex-panel.js:18-30):

<Steps>
  <Step title="OAuth Setup">
    First runs `setup-oauth.js` to configure HTTP routes
    
    ```javascript
    const setupOAuth = spawn('node', [setupOAuthPath], { 
      stdio: 'inherit', 
      cwd: process.cwd() 
    });
    ```
  </Step>

  <Step title="Environment Setup">
    On successful OAuth setup, runs `setup-env.js` to configure environment variables
    
    ```javascript
    setupOAuth.on('close', (code) => {
      if (code !== 0) process.exit(code);
      const setupEnv = spawn('node', [setupEnvPath], { 
        stdio: 'inherit', 
        cwd: process.cwd() 
      });
    });
    ```
  </Step>
</Steps>

<Note>
Both scripts must complete successfully. If OAuth setup fails, environment setup won't run.
</Note>

## OAuth Setup Details

### File Detection

The OAuth setup script intelligently detects your working directory (setup-oauth.js:25-46):

```javascript
const currentDirName = path.basename(currentDir);

if (currentDirName === "convex") {
  // Work with http.ts directly
  httpPath = path.join(currentDir, "http.ts");
} else {
  // Work with convex/http.ts from project root
  httpPath = path.join(currentDir, "convex", "http.ts");
}
```

### Route Registration

The script registers both routes before the `export default http;` statement (setup-oauth.js:359-369):

```typescript
http.route({
  path: "/oauth/exchange",
  method: "POST",
  handler: exchangeOAuthCode,
});

http.route({
  path: "/oauth/exchange",
  method: "OPTIONS",
  handler: preflightOAuthExchange,
});
```

### Idempotency

The setup is safe to run multiple times (setup-oauth.js:86-91):

```javascript
if (httpContent.includes(OAUTH_MARKER) || httpContent.includes(ROUTE_MARKER)) {
  console.log(
    "[convex-panel] OAuth HTTP action already present in convex/http.ts; no changes made."
  );
  return;
}
```

## Advanced Usage

### Running from Project Root

```bash
cd my-convex-project
convex-panel setup
```

The script will create or modify `convex/http.ts` in your project.

### Running from Convex Directory

```bash
cd my-convex-project/convex
convex-panel setup
```

The script will create or modify `http.ts` in the current directory.

### Re-running Setup

You can safely re-run setup at any time:

```bash
convex-panel setup
```

<Info>
The setup is idempotent - it won't duplicate OAuth routes or overwrite existing environment variables.
</Info>

## Environment File Priority

The setup script follows this priority order (setup-env.js:378-389):

1. `.env.local` (if exists)
2. `.env` (if exists)
3. Creates `.env.local` (if neither exists)

```javascript
async function pickEnvFile(cwd) {
  const candidates = [".env.local", ".env"];
  for (const name of candidates) {
    const full = path.join(cwd, name);
    try {
      await fs.access(full);
      return full;
    } catch {}
  }
  return path.join(cwd, ".env.local");
}
```
