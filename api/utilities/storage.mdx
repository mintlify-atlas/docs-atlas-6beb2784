Utilities for managing local storage with built-in in-memory caching for optimal performance. These functions handle storage operations for filters, table state, and user preferences.

## Core Storage Functions

### getStorageItem

Retrieves a value from localStorage with automatic in-memory caching.

```typescript
function getStorageItem<T>(key: string, defaultValue: T): T
```

<ParamField path="key" type="string" required>
  Storage key to retrieve
</ParamField>

<ParamField path="defaultValue" type="T" required>
  Default value to return if key doesn't exist
</ParamField>

**Returns:** The stored value of type `T`, or `defaultValue` if not found

<Card title="Example" icon="code">
```typescript
import { getStorageItem } from 'convex-panel';

const theme = getStorageItem('app-theme', 'light');
console.log('Current theme:', theme);

const settings = getStorageItem('user-settings', {
  notifications: true,
  darkMode: false
});
```
</Card>

<Note>
  Values are automatically cached in memory after first retrieval. Subsequent calls return the cached value without reading from localStorage.
</Note>

### setStorageItem

Stores a value in localStorage and updates the in-memory cache.

```typescript
function setStorageItem<T>(key: string, value: T): void
```

<ParamField path="key" type="string" required>
  Storage key
</ParamField>

<ParamField path="value" type="T" required>
  Value to store (will be JSON serialized)
</ParamField>

<Card title="Example" icon="code">
```typescript
import { setStorageItem } from 'convex-panel';

setStorageItem('app-theme', 'dark');

setStorageItem('user-settings', {
  notifications: false,
  darkMode: true
});
```
</Card>

### removeStorageItem

Removes a value from localStorage.

```typescript
function removeStorageItem(key: string): void
```

<ParamField path="key" type="string" required>
  Storage key to remove
</ParamField>

<Card title="Example" icon="code">
```typescript
import { removeStorageItem } from 'convex-panel';

removeStorageItem('temporary-data');
```
</Card>

## Table Management

### getTableFilters

Retrieves stored filters for a specific table with optimized caching.

```typescript
function getTableFilters(tableName: string): {
  clauses: FilterClause[];
}
```

<ParamField path="tableName" type="string" required>
  Name of the table
</ParamField>

**Returns:** Object containing filter clauses array

<Card title="Example" icon="code">
```typescript
import { getTableFilters } from 'convex-panel';

const filters = getTableFilters('users');
console.log('Active filters:', filters.clauses);

// filters = {
//   clauses: [
//     { field: 'status', operator: 'eq', value: 'active' },
//     { field: 'age', operator: 'gte', value: 18 }
//   ]
// }
```
</Card>

### saveTableFilters

Saves filters for a specific table with optimized caching.

```typescript
function saveTableFilters(
  tableName: string,
  filters: { clauses: FilterClause[] }
): void
```

<ParamField path="tableName" type="string" required>
  Name of the table
</ParamField>

<ParamField path="filters" type="{clauses: FilterClause[]}" required>
  Filter configuration object
</ParamField>

<Card title="Example" icon="code">
```typescript
import { saveTableFilters } from 'convex-panel';

saveTableFilters('users', {
  clauses: [
    { field: 'status', operator: 'eq', value: 'active' },
    { field: 'age', operator: 'gte', value: 18 }
  ]
});
```
</Card>

<Note>
  If the clauses array is empty, the table entry is removed from storage completely to keep storage clean.
</Note>

<ParamField path="tableName" type="string" required>

## Recently Viewed

### getRecentlyViewedTables

Retrieves the list of recently viewed tables.

```typescript
function getRecentlyViewedTables(): Array<{
  name: string;
  timestamp: number;
}>
```

**Returns:** Array of recently viewed table objects, sorted by most recent

<Card title="Example" icon="code">
```typescript
import { getRecentlyViewedTables } from 'convex-panel';

const recentTables = getRecentlyViewedTables();
recentTables.forEach(table => {
  const date = new Date(table.timestamp);
  console.log(`${table.name} - last viewed ${date.toLocaleString()}`);
});
```
</Card>

### updateRecentlyViewedTable

Adds or updates a table in the recently viewed list.

```typescript
function updateRecentlyViewedTable(tableName: string): void
```

<ParamField path="tableName" type="string" required>
  Name of the table to add to recently viewed
</ParamField>

<Card title="Example" icon="code">
```typescript
import { updateRecentlyViewedTable } from 'convex-panel';

// When user views a table
function onTableView(tableName: string) {
  updateRecentlyViewedTable(tableName);
  // Table is now at the top of recently viewed list
}
```
</Card>

<Note>
  - If the table already exists in the list, it's moved to the top with an updated timestamp
  - Only the 10 most recent tables are kept
  - Older tables are automatically removed
</Note>

## Storage Cleanup

### clearAllStorage

Clears all Convex Panel data from storage (for debugging or reset).

```typescript
function clearAllStorage(): void
```

<Warning>
  This function removes ALL Convex Panel data including:
  - OAuth tokens
  - Table filters and sort configurations
  - Recently viewed tables
  - User preferences
  - All cached data
  
  Use with caution!
</Warning>

<Card title="Example" icon="code">
```typescript
import { clearAllStorage } from 'convex-panel';

function handleReset() {
  if (confirm('Clear all Convex Panel data?')) {
    clearAllStorage();
    window.location.reload();
  }
}
```
</Card>

## Complete Table State Example

<Card title="Example: Managing Table State" icon="table">
```typescript
import {
  getActiveTable,
  saveActiveTable,
  getTableFilters,
  saveTableFilters,
  getTableSortConfig,
  saveTableSortConfig,
  updateRecentlyViewedTable
} from 'convex-panel';

function TableManager() {
  // Get current table
  const currentTable = getActiveTable();
  
  // Switch to a different table
  function switchTable(tableName: string) {
    // Save as active
    saveActiveTable(tableName);
    
    // Update recently viewed
    updateRecentlyViewedTable(tableName);
    
    // Load table state
    const filters = getTableFilters(tableName);
    const sortConfig = getTableSortConfig(tableName);
    
    console.log('Switched to table:', tableName);
    console.log('Filters:', filters);
    console.log('Sort:', sortConfig);
  }
  
  // Apply filters to current table
  function applyFilters(filters: any) {
    const tableName = getActiveTable();
    if (tableName) {
      saveTableFilters(tableName, filters);
      console.log('Filters applied');
    }
  }
  
  // Apply sort to current table
  function applySort(field: string, direction: 'asc' | 'desc') {
    const tableName = getActiveTable();
    if (tableName) {
      saveTableSortConfig(tableName, { field, direction });
      console.log(`Sorted by ${field} ${direction}`);
    }
  }
  
  return {
    currentTable,
    switchTable,
    applyFilters,
    applySort
  };
}

// Usage
const manager = TableManager();
manager.switchTable('users');
manager.applyFilters({
  clauses: [
    { field: 'status', operator: 'eq', value: 'active' }
  ]
});
manager.applySort('createdAt', 'desc');
```
</Card>

## Storage Keys

All Convex Panel storage keys use the prefix `convex-panel:` to avoid conflicts. Common keys include:

- `convex-panel:oauth-token` - OAuth access token
- `convex-panel:active-table` - Currently active table
- `convex-panel:table-filters` - All table filters
- `convex-panel:recently-viewed-tables` - Recently viewed tables list
- `convex-panel:logs-filters` - Logs view filters
- `convex-panel:theme` - UI theme preference

## Performance Notes

<Note>
  **In-Memory Caching**: All storage operations use an in-memory cache to minimize localStorage reads. The first access reads from localStorage, subsequent accesses use the cached value.
  
  **Write-Through**: Writes update both the memory cache and localStorage simultaneously for consistency.
  
  **SSR Safety**: All functions check for `window` existence and gracefully handle server-side rendering scenarios.
</Note>
