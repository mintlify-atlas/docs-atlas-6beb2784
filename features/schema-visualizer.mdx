The Schema Visualizer provides an interactive, visual representation of your Convex database structure, making it easy to understand table relationships and design patterns.

## Overview

Visualize your entire database schema as an interactive diagram. See tables, fields, relationships, and indexes at a glance. The Schema Visualizer uses ReactFlow to create dynamic, explorable diagrams of your data model.

<Note>
  Access the Schema Visualizer from the Data Browser by clicking the schema icon in the toolbar.
</Note>

### Table Nodes

Each table displays:

```
┌─────────────────────────────┐
│  📊 messages                │
├─────────────────────────────┤
│  _id: Id<"messages">        │
│  _creationTime: number      │
│  userId: Id<"users">    ━━━ │──→ (link to users table)
│  text: string               │
│  channel: string            │
│  metadata?: object          │
├─────────────────────────────┤
│  📑 Indexes: 2              │
│  📝 Documents: 1,234        │
└─────────────────────────────┘
```

<Tabs>
  <Tab title="Fields">
    **Field Information**
    
    Each field shows:
    - **Field name**: The property name
    - **Type**: TypeScript/Convex type
    - **Optional marker**: `?` for optional fields
    - **Reference indicator**: Links to other tables
    
    <Tip>
      Click a field name to see its full type definition.
    </Tip>
  </Tab>
  
  <Tab title="Relationships">
    **Visual Connections**
    
    Relationships shown as arrows:
    - **Solid line**: Direct reference (`Id<"table">`)
    - **Dashed line**: Inferred relationship
    - **Arrow direction**: From referencing to referenced
    - **Bidirectional**: When both tables reference each other
    
    ```
    messages ━━━━━━→ users
           ←━━━━━━━ (userId field)
    ```
  </Tab>
  
  <Tab title="Indexes">
    **Index Indicators**
    
    See which fields are indexed:
    - Count of indexes on table
    - Click to view index details
    - Index names and field combinations
    - Unique vs non-unique indexes
  </Tab>
  
  <Tab title="Stats">
    **Table Statistics**
    
    Quick metrics for each table:
    - Document count
    - Field count
    - Index count
    - Last modified time
  </Tab>
</Tabs>

## Layout Algorithms

Choose the best layout for your schema:

<CardGroup cols={3}>
  <Card title="Force-Directed" icon="magnet">
    Physics-based automatic layout
    - Tables repel each other
    - Relationships pull tables together
    - Creates organic, readable layouts
    - Best for: Medium schemas (5-20 tables)
  </Card>
  
  <Card title="Hierarchical" icon="sitemap">
    Tree-like top-down structure
    - Parent tables at top
    - Child tables below
    - Clear dependency flow
    - Best for: Clear hierarchies
  </Card>
  
  <Card title="Radial" icon="circle-nodes">
    Circular arrangement
    - Core tables in center
    - Related tables around edges
    - Emphasizes central entities
    - Best for: Star schemas
  </Card>
</CardGroup>

<Tip>
  Try different layouts to find the clearest view for your schema. Your layout choice is saved per project.
</Tip>

## Export Options

Save your schema diagrams:

<Tabs>

    - Includes current zoom/pan view
    - Perfect for documentation
    
    Use cases:
    - README files
    - Wiki pages  
    - Presentations
    - Team documentation
  </Tab>
  
  <Tab title="SVG Export">
    **Vector Graphic**
    
    Download as SVG file:
    - Infinite scaling without quality loss
    - Editable in design tools
    - Smaller file size
    - Web-friendly format
    
    Use cases:
    - Technical diagrams
    - Print materials
    - Design mockups
    - Interactive docs
  </Tab>
  
  <Tab title="JSON Export">
    **Schema Data**
    
    Export schema as JSON:
    - Complete table definitions
    - Field types and validators
    - Index configurations
    - Programmatic processing
    
    ```json
    {
      "tables": {
        "messages": {
          "fields": {
            "userId": "Id<users>",
            "text": "string"
          },
          "indexes": ["by_user", "by_channel"]
        }
      }
    }
    ```
  </Tab>
</Tabs>

### Relationship Patterns

Identify common database patterns:

- **One-to-many**: Single parent, multiple children
- **Many-to-many**: Junction tables connecting entities
- **Hierarchical**: Parent-child trees
- **Hub-and-spoke**: Central table with many references

## Troubleshooting

<AccordionGroup>
  <Accordion title="Diagram not loading">
    - Check that tables exist in deployment
    - Verify schema is defined in convex/schema.ts
    - Try refreshing the page
    - Check browser console for errors
  </Accordion>
  
  <Accordion title="Layout looks messy">
    - Try a different layout algorithm
    - Manually arrange key tables, then re-run layout
    - Use focus mode to view subsets
    - Zoom in to see detail
  </Accordion>
  
  <Accordion title="Relationships not showing">
    - Ensure you're using `Id<"tableName">` types
    - Check that schema includes relationship fields
    - Try toggling "Show inferred relationships"
    - Verify field types are properly defined
  </Accordion>
</AccordionGroup>
