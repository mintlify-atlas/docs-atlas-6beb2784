The Health Metrics view provides real-time monitoring of your Convex deployment's performance, errors, and resource usage.

## Overview

Health Metrics gives you a comprehensive dashboard of your application's vital signs. Monitor function performance, track errors, analyze cache efficiency, and identify issues before they impact users.

<Tip>
  All metrics are displayed for the last hour by default and update automatically every 30 seconds.
</Tip>

## Performance Charts

### Slowest Functions

Identify performance bottlenecks:

<Tabs>
  <Tab title="p50 Latency">
    **Median Execution Time**
    
    View the median (50th percentile) execution time for each function:
    - Bar chart visualization
    - Sorted by slowest first
    - Compare across functions
    - Click to view function details
    
    <Note>
      p50 represents typical performance - half of executions are faster, half are slower.
    </Note>
  </Tab>
  
  <Tab title="p95 Latency">
    **95th Percentile**
    
    See worst-case performance:
    - 95% of executions faster than this
    - Catch outliers and slow queries
    - Identify optimization opportunities
  </Tab>
  
  <Tab title="p99 Latency">
    **99th Percentile**
    
    Monitor edge cases:
    - Nearly all executions faster
    - Spot rare performance issues
    - Track timeout risks
  </Tab>
</Tabs>

## Error Tracking

### Top Errors

Monitor the most frequent errors:

<AccordionGroup>
  <Accordion title="Error Overview">
    - **Error message**: The exception or error text
    - **Occurrences**: How many times seen
    - **First seen**: When error first appeared
    - **Last seen**: Most recent occurrence
    - **Affected functions**: Which functions throw this error
  </Accordion>
  
  <Accordion title="Error Grouping">
    Errors are automatically grouped by:
    - Error message similarity
    - Stack trace patterns
    - Function name
    - Error type (TypeError, ReferenceError, etc.)
  </Accordion>
  
  <Accordion title="Error Details">
    Click an error to view:
    - Full stack trace
    - Recent occurrences timeline
    - Affected request IDs
    - Link to logs for debugging
    - AI-generated fix suggestions
  </Accordion>
</AccordionGroup>

## Monitoring Workflows

### Daily Health Check

<Steps>
  <Step title="Check failure rate">
    Ensure it's below your threshold (typically less than 1%)
  </Step>
  <Step title="Review top errors">
    Prioritize fixing most frequent errors
  </Step>
  <Step title="Check slowest functions">
    Identify optimization opportunities
  </Step>
  <Step title="Review insights">
    Act on AI-generated recommendations
  </Step>
</Steps>

### Post-Deployment Validation

<Steps>
  <Step title="Compare before/after metrics">
    Check if performance improved or regressed
  </Step>
  <Step title="Watch for new errors">
    Monitor error count for 15-30 minutes
  </Step>
  <Step title="Verify cache hit rate">
    Ensure caching still working after changes
  </Step>
  <Step title="Check function latencies">
    Confirm no unexpected slowdowns
  </Step>
</Steps>

### Incident Response

<Steps>
  <Step title="Identify spike">
    Check failure rate and error charts
  </Step>
  <Step title="Find affected functions">
    Look at top errors and slowest functions
  </Step>
  <Step title="View recent logs">
    Switch to Logs view for details
  </Step>
  <Step title="Check deployment correlation">
    See if issue started with a deployment
  </Step>
</Steps>
