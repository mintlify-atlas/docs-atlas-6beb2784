The File Storage view provides a comprehensive interface for managing files stored in Convex, including images, documents, videos, and other media.

## Overview

Manage all files uploaded to your Convex deployment through the File Storage interface. Upload new files via drag-and-drop, preview images and documents, and efficiently manage your file library.

## File Metadata

Each file stores:

```typescript
interface FileMetadata {
  _id: string;              // Document ID
  _creationTime: number;    // Upload timestamp
  storageId: string;        // Storage ID for retrieval
  name?: string;            // Original filename
  size?: number;            // File size in bytes
  contentType?: string;     // MIME type
  url?: string;            // Signed URL for access
}
```

## Best Practices

<AccordionGroup>
  <Accordion title="Optimize file sizes">
    - Compress images before upload
    - Use appropriate formats (WebP for images, MP4 for video)
    - Don't store unnecessarily large files
    - Consider resizing images server-side
  </Accordion>
  
  <Accordion title="Use descriptive names">
    ```typescript
    // Good
    await storage.store(file, "user-avatar-123.jpg");
    
    // Avoid
    await storage.store(file, "image.jpg");
    ```
  </Accordion>
  
  <Accordion title="Implement cleanup">
    Set up scheduled jobs to delete old files:
    ```typescript
    export const cleanupOldFiles = internalMutation({
      handler: async (ctx) => {
        const thirtyDaysAgo = Date.now() - 30 * 24 * 60 * 60 * 1000;
        const oldFiles = await ctx.db.query("_storage")
          .filter(q => q.lt(q.field("_creationTime"), thirtyDaysAgo))
          .collect();
        // Delete old files
      }
    });
    ```
  </Accordion>
  
  <Accordion title="Track file ownership">
    Store references to files in your data model:
    ```typescript
    export default defineSchema({
      users: defineTable({
        name: v.string(),
        avatarStorageId: v.optional(v.id("_storage")),
      }),
    });
    ```
  </Accordion>
</AccordionGroup>

## Integration with Code

### Upload from Functions

```typescript
import { mutation } from "./_generated/server";

export const uploadFile = mutation({
  handler: async (ctx, { file }: { file: File }) => {
    const storageId = await ctx.storage.store(file);
    
    // Save reference in your table
    await ctx.db.insert("documents", {
      title: file.name,
      storageId,
      size: file.size,
      contentType: file.type,
    });
    
    return storageId;
  },
});
```

### Download from Functions

```typescript
import { query } from "./_generated/server";

export const getFileUrl = query({
  args: { storageId: v.id("_storage") },
  handler: async (ctx, { storageId }) => {
    return await ctx.storage.getUrl(storageId);
  },
});
```

### Delete from Functions

```typescript
import { mutation } from "./_generated/server";

export const deleteFile = mutation({
  args: { storageId: v.id("_storage") },
  handler: async (ctx, { storageId }) => {
    await ctx.storage.delete(storageId);
  },
});
```

## Troubleshooting

<AccordionGroup>
  <Accordion title="Upload fails">
    - Check file size against plan limits
    - Verify network connection
    - Try a different file format
    - Check browser console for errors
    - Ensure you have write permissions
  </Accordion>
  
  <Accordion title="Can't preview file">
    - Some file types don't support preview
    - Try downloading and opening locally
    - Check file isn't corrupted
    - Verify content type is set correctly
  </Accordion>
  
  <Accordion title="Files not showing">
    - Check component filter
    - Verify deployment connection
    - Try refreshing the page
    - Check date filter isn't too restrictive
  </Accordion>
</AccordionGroup>
