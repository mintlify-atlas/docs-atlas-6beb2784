The Function Runner provides an interactive environment for testing, debugging, and analyzing your Convex queries, mutations, and actions.

## Overview

Test your Convex functions without writing frontend code. The Function Runner combines a code editor, execution environment, and performance analyzer in one interface.

## Key Features

### Function Browser

Navigate your function library:

  <Tab title="By File">

    - File path breadcrumbs

    ```
    📁 convex/
    ├── 📄 messages.ts
    │   ├── fn send (mutation)

    │   └── fn delete (mutation)
    └── 📄 users.ts
        ├── fn get (query)
    ```

  </Tab>
  
  <Tab title="By Type">
    **Function Type Badges**
    
    Visual indicators for function types:
    
    <CardGroup cols={3}>
      <Card title="Query" icon="Q">
        🟡 Read-only database queries
      </Card>

    </CardGroup>
  </Tab>
  
  <Tab title="By Component">

    Filter functions by Convex component:
    - Main app functions

### Function Execution

Run functions with custom arguments:

<Steps>
  <Step title="Select function">
    Click any function in the sidebar to load it
  </Step>
  <Step title="Enter arguments">
    Provide input in JSON format:
    ```json
    {
      "userId": "user123",
      "message": "Hello world",
      "metadata": { "priority": "high" }
    }
    ```
  </Step>
  <Step title="Execute">
    Click "Run Function" or press Cmd/Ctrl+Enter
  </Step>
  <Step title="View results">
    See return value, logs, and performance metrics
  </Step>
</Steps>

<Tip>
  Arguments are saved per function so you can quickly re-run tests.
</Tip>

### Interactive Code Editor

Full-featured Monaco editor with:

- **Syntax highlighting**: TypeScript/JavaScript support
- **Auto-completion**: IntelliSense for Convex APIs
- **Error detection**: Real-time linting
- **Formatting**: Prettier integration
- **Dark/light themes**: Matches your panel theme

### Results Panel

View detailed execution results:

<Tabs>
  <Tab title="Return Value">
    **Function Output**
    
    See the returned data with formatting:
    - Syntax-highlighted JSON
    - Collapsible nested objects
    - Copy to clipboard
    - Type indicators
    
    ```json
    {
      "_id": "msg_abc123",
      "_creationTime": 1234567890,
      "text": "Hello world",
      "author": {
        "_id": "user_xyz",
        "name": "Alice"
      }
    }
    ```
  </Tab>
  
  <Tab title="Console Logs">
    **Debug Output**
    
    View console.log statements from your function:
    - All log levels (debug, info, warn, error)
    - Timestamps for each log
    - Formatted objects and arrays
    - Click to expand complex values
    
    <Note>
      Logs appear in real-time as the function executes.
    </Note>
  </Tab>
  
  <Tab title="Errors">
    **Exception Details**
    
    When functions fail, see:
    - Full error message
    - Stack trace with line numbers
    - Click to jump to error in code
    - AI-suggested fixes
    
    ```typescript
    Error: User not found
        at Object.handler (users.ts:42:11)
        at processTicksAndRejections (internal/process/task_queues.js:95:5)
    ```
  </Tab>
  
  <Tab title="Performance">
    **Execution Metrics**
    
    Analyze function performance:
    - **Duration**: Total execution time
    - **Database ops**: Read/write counts
    - **Memory**: MB consumed
    - **Cache**: Whether result was cached
    - **Compute**: GB-hours calculation
  </Tab>
</Tabs>

### Logs Tab

Stream real-time function execution logs:

- **Live streaming**: See executions as they happen
- **Request ID**: Unique identifier for each execution
- **Timestamp**: When the function ran
- **Status**: Success, failure, or cached
- **Duration**: Execution time
- **Click to expand**: View full details for any execution

<Note>
  The Logs tab shows only logs for the selected function, filtered from the global log stream.
</Note>

```typescript
const results = await db.query("messages")

  .take(20);
```

Test functions with multiple input sets:

```json
// Test case 1

// Test case 2
```

## Debugging Workflows

### Testing a New Function

<Steps>
  <Step title="Navigate to function">
    Find your new function in the sidebar
  </Step>
  <Step title="Prepare test data">
    Enter realistic arguments
  </Step>
  <Step title="Run and inspect">
    Execute and check return value
  </Step>
  <Step title="Test edge cases">
    Try invalid inputs, empty data, etc.
  </Step>
  <Step title="Monitor performance">
    Check execution time and resource usage
  </Step>
</Steps>

### Debugging an Error

<Steps>
  <Step title="Reproduce the error">
    Use arguments from production logs
  </Step>
  <Step title="Examine stack trace">
    Identify the failing line of code
  </Step>
  <Step title="View source code">
    Switch to Code tab to see context
  </Step>
  <Step title="Add debug logs">
    Deploy with additional console.log
  </Step>
  <Step title="Re-test">
    Run function again and check logs
  </Step>
</Steps>

### Performance Optimization

<Steps>
  <Step title="Establish baseline">
    Run function and note execution time
  </Step>
  <Step title="Check database ops">
    See how many reads/writes occur
  </Step>
  <Step title="Optimize code">
    Add indexes, reduce queries, etc.
  </Step>
  <Step title="Deploy changes">
    Push optimized code
  </Step>
  <Step title="Compare results">
    Re-run and measure improvement
  </Step>
</Steps>

## Best Practices

<AccordionGroup>
  <Accordion title="Save test cases">
    Keep a file of common test arguments for quick testing:
    ```json
    // test-data.json
    {
      "sendMessage": {
        "userId": "test_user",
        "text": "Test message"
      },
      "getUser": {
        "userId": "test_user"
      }
    ```

  </Accordion>
  
  <Accordion title="Test before deploying">

  </Accordion>
  
  <Accordion title="Use realistic data">
    Test with production-like data sizes and structures

## Troubleshooting

<AccordionGroup>
  <Accordion title="Function won't execute">
    - Check argument syntax is valid JSON
    - Verify you have correct permissions
    - Ensure deployment is connected
    - Check function is deployed
  </Accordion>
  
  <Accordion title="Can't see source code">
    - Verify you're connected to the right deployment
    - Check the function exists in current deployment
    - Try refreshing the page
    - Ensure deployment has source maps
  </Accordion>
  
  <Accordion title="Performance metrics missing">
    - Metrics require at least one function execution
    - Wait 30 seconds for chart updates
    - Check deployment connection
    - Verify function has been called recently
  </Accordion>
</AccordionGroup>
